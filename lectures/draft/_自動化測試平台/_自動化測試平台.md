##核心是：不要把 command 等同於 SCPI 字串。

Test Step
  ↓
Logical Command
  ↓
Protocol Adapter
  ↓
Physical Transport

也就是：

set_voltage
  → SCPI Adapter
  → "VOLT 12"

read_status
  → Binary Adapter
  → 0xA5 0x01 0x02 CRC

read_register
  → Modbus Adapter
  → RTU/TCP frame
  
##新增三層抽象
1. Instrument Driver

3. Transport Layer

##測試腳本 = 動態流程 + 邏輯命令 + 多協定轉換 + 多儀器協同

所以 Web 端不要只設計「SCPI Command Table」，而要設計成：

Command Definition
  command_name
  target_role
  protocol_type
  request_encoder
  response_decoder
  timeout
  retry
  output_schema
  
  
#結論

Web Server：
  管理測試流程、命令定義、協定定義、版本、報告

Local Python Worker：
  負責動態流程執行、多儀器連線、Protocol Adapter、Transport Driver

Test Package：
  不是單純 script，而是完整測試描述包

Command：
  不是 SCPI 字串，而是 logical command

一句話：
Web 端產生「測試描述」，Worker 端用「多協定執行引擎」跑測試。
